Sorry, this is our fault. The replacement length is set incorrectly when adding imports.
I'll provide a patch
Created attachment 76538
fix

1. Do not rewrite document if no imports to add
2. Do not set the completion length when adding imports:
    - Setting the length has bad side effects on a position which is shared in DocumentTemplateContext
    - Inserting an import only changes the completion offset, never the length anyway

Btw, even with 1 fixed the pattern:

List ${list:newName(java.util.List)}= ${word_selection};
ArrayList ${arrayList:newName(java.util.ArrayList)};

can be used to reproduce
