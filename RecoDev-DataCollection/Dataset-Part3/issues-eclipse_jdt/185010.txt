Annotation processors are modules of third-party, relatively untrusted, code that run in the context of the compiler.  We attempt to catch and gracefully handle all exceptions thrown by processors.  To this end, there is a "try / catch (Throwable)" block in AptCompilationParticipant.reconcile().

However, this is insufficient, because there is also a try / catch (Exception) block in CompilationUnitResolver.resolve(), which is called underneath reconcile.  Instead, we need to push the try/catch further down, e.g. to the ReconcileEnv callback:
