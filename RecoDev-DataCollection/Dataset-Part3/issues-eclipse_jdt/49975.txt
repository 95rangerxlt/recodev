I analyzed the problem and can confirm the following behavior: there's a
BestMatchHover (BMH) per partition type (e.g. Java code, Java comment,...) and
per editor. Once the BMH is created for the given partition type it will no
longer go through that code. Hence it is not that bad since once created the
hovers are shown without going through that code again. Typing, switching to
other editors and such do not affect that behavior.

I have now changed the code to cache that information so that it is computed
only once per session. I.e. using N20040116 or newer you should get exactly one
(long) print out.
