Created attachment 8573890
responsive-design-intermittent.patch

Here's a patch that should fix this intermittent. The resize events are apparently suppressed during document load. A bit of logging revealed that when the test failed, the document finished loading only after the viewport size was already set causing the resize event to be missing and thus a timeout. This happens intermittently as the test sets document.location to new value and then waits for next tick which may or may not give the document enough time to complete loading.

This patch borrows method waitForDocLoadComplete from [1] and makes the test wait for the document to be loaded before the view is resized the first time. It seems to work locally: without this patch about a timeout occurred about every 3-5 runs and with this patch it completed 200 times without a single timeout.
