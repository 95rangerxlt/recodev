Comment on attachment 206366

We should remove the assertion, and instead of using NS_ENSURE_TRUE, we should use if ()
The reason is that it's actually fairly normal for this to happen. If a DOM node goes away inside Mozilla, but the AT asks for info on it, we hold onto the accessible, but mDOMNode is nsnull. Unfortunately NS_ENSURE_TRUE prints debugging messages in the console.

For an example see:
http://lxr.mozilla.org/seamonkey/source/accessible/src/base/nsAccessible.cpp#185
